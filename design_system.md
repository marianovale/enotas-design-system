# eNotas Design System — Design Context

> Extracted once from Figma + implemented tokens. Reuse as source of truth for all AI-assisted UI work.
> Do NOT call the Figma API again unless tokens or components change significantly.
> CSS prefix: `--en-*`

---

# 0. Overview

**Purpose**
Consistent, scalable UI system for eNotas (Hotmart). Delivered as Web Components (Stencil.js) that run natively in ASP.NET Razor Views + jQuery — no React required today. Future path: merge with Hotmart Cosmos DS via React wrappers auto-generated by Stencil.

**Scope**
- Platforms: Web (desktop-first)
- Stack: Web Components (`<en-*>` tags), CSS Custom Properties (`--en-*`)
- Coverage: Foundations · 30 Components · NF-specific patterns

**Usage in HTML / Razor**
```html
<script type="module" src="/dist/enotas-ds/enotas-ds.esm.js"></script>
<link rel="stylesheet" href="/dist/tokens.css">
<!-- then use anywhere: -->
<en-button variant="primary" size="md">Emitir Nota</en-button>
```

# 2. Components

All components are Web Components with tag prefix `en-`. Use as HTML attributes. Array/object props must be set via JavaScript (`.propName = value`).

Events are prefixed `en` (e.g. `enClick`, `enChange`). Work natively with jQuery: `$('en-button').on('enClick', fn)`.

# 5. Accessibility

- Contrast ratio ≥ 4.5:1 (text vs background)
- All interactive elements have visible focus ring (`--en-border-focus: #22baa0`, `border-medium: 2px`)
- Keyboard navigation: Tab · Enter · Esc · Arrow keys (Popover, Modal, Tour)
- ARIA: `role`, `aria-label`, `aria-disabled`, `aria-expanded` on interactive components

---

# 6. Rules

- **Always use `--en-*` tokens** — never raw hex or pixel values
- **Array/object props** must be set via JS (`.options`, `.items`, `.steps`, `.value`)
- **Events** are always prefixed `en` — use `addEventListener('enClick', fn)` or `$().on('enClick', fn)`
- One semantic token per purpose — do not reuse `action` tokens for `feedback` contexts
- `variant="negative"` on `<en-button>` for destructive actions (delete, cancel NF) — `color="danger"` is deprecated
- Use `<en-spinner loading>` inside `<en-button loading>` — do not create custom loaders
